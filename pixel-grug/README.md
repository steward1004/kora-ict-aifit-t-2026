# Pixel Grug — Landing Page

> **Tiny Pixel Stories for Better Days.**
> 매일 한 문장, 한 개의 픽셀 캐릭터, 그리고 작은 미소.

AI가 매일 새로운 픽셀 친구를 만들어 주는 감성 마이크로 저널 앱 **Pixel Grug**의 제품 소개 랜딩페이지입니다. 순수 HTML/CSS/JS로 제작되었으며, **다국어(한국어 / 영어 / 일본어)** 와 **라이트 / 다크 테마(Anthropic Light & Dark Theme)** 를 지원합니다.

## 미리보기

![Pixel Grug 히어로 캐릭터](assets/hero-dragon.png)

## 기술 스택

- **HTML5** — 시맨틱 마크업, `data-i18n` 기반 다국어 속성
- **CSS3** — CSS 커스텀 프로퍼티(디자인 토큰), 테마 스위칭(`[data-theme]`), Grid/Flex 레이아웃, 반응형
- **Vanilla JavaScript** — 프레임워크 없는 순수 인터랙션, 경량 i18n 엔진, 테마/언어 상태 관리
- **Google Fonts** — Fraunces(세리프), Inter(산세리프), Press Start 2P(픽셀)
- **AI 픽셀아트 에셋** — 생성형 이미지로 캐릭터/오브젝트 제작

## 디자인 (Anthropic Light & Dark Theme)

CSS 커스텀 프로퍼티로 디자인 토큰을 정의하고, `<html data-theme="...">` 값에 따라 라이트/다크 팔레트를 전환합니다.

| 항목 | 라이트 | 다크 |
| --- | --- | --- |
| 배경 | 웜 아이보리 `#F0EEE6` / 카드 `#FAF9F5` | 웜 다크 `#1F1E1C` / 카드 `#262523` |
| 텍스트 | 잉크 `#1A1915` / 뮤트 `#6B675D` | `#F2F0E9` / 뮤트 `#A39E91` |
| 액센트 | 클레이·코랄 `#CC785C` | 밝은 클레이 `#D68A6F` |
| 폰트 | 헤딩 Fraunces(serif) · 본문 Inter(sans) · 강조 Press Start 2P(pixel) | (공통) |
| 스타일 | 넉넉한 여백, 소프트 라운드, 미묘한 그림자 | (공통) |

> 인버스 패널(핵심 컨셉·비전·셰어 카드), 이미지 프레임, 네비 배경, CTA 버튼은 별도 토큰(`--panel-*`, `--frame-bg`, `--nav-bg`, `--cta-bg`)으로 분리하여 두 테마 모두에서 올바르게 표시됩니다.

## 다국어 & 테마 지원

- **다국어 (한국어 / 영어 / 일본어)**
  - 모든 텍스트에 `data-i18n` 키를 부여하고, `js/main.js`의 `I18N` 사전에서 언어별 문자열을 주입합니다.
  - 마크업 포함 텍스트는 `data-i18n-html`, 메타 태그는 `data-i18n-attr`로 처리합니다.
  - 네비의 **`KO / EN / 日` 세그먼트 토글**로 전환하며, 선택값은 `localStorage`(`pg-lang`)에 저장됩니다. 최초 방문 시 브라우저 언어를 자동 감지합니다.
  - 전환 시 `<html lang>` 속성과 문서 제목(`<title>`)도 함께 갱신됩니다.
- **라이트 / 다크 테마**
  - 네비의 **☀ / ☾ 테마 토글**로 전환하며, 선택값은 `localStorage`(`pg-theme`)에 저장됩니다.
  - 최초 방문 시 OS의 `prefers-color-scheme`를 따르고, `<head>`의 인라인 스크립트로 깜빡임(FOUC) 없이 초기 테마를 적용합니다.

## 인터랙션 (JavaScript)

- **언어 토글** — `KO / EN / 日` 전환, `localStorage` 저장, 브라우저 언어 자동 감지
- **테마 토글** — 라이트/다크 전환, `localStorage` 저장, `prefers-color-scheme` 기본값
- **모바일 햄버거 메뉴** — 토글 + 접근성(`aria-expanded`) 속성
- **스크롤 리빌** — `IntersectionObserver` 기반 페이드/슬라이드 등장
- **네비 스크롤 상태** — 스크롤 위치에 따른 배경/그림자 전환
- **Daily Pixel 데모** — "다른 친구 만나기" 버튼으로 6종 캐릭터/문장 순환 전환 (언어별 이름 현지화, 이미지 프리로드 포함)

## 개발 과정 및 오류 수정 사항

작업 중 발견하여 수정한 주요 이슈입니다.

- **모바일 히어로 카드 가로 오버플로우 수정**
  - 증상: 모바일 뷰에서 "오늘의 픽셀" 카드(고정 `max-width: 360px`)가 컨테이너 가용 폭(약 342px)보다 넓어, CSS Grid 트랙이 확장되면서 카드 우측이 뷰포트 밖으로 잘림.
  - 원인: Grid 아이템의 자동 최소 크기(`min-width: auto`)가 카드의 고정 폭을 따라가며 트랙이 컨테이너를 초과.
  - 해결: 모바일 브레이크포인트에서 `.hero__visual { min-width: 0 }`, `.daily-card { max-width: 100% }`를 적용하여 카드가 컨테이너 안에 맞도록 수정.

- **모바일 타이포/패딩 정리**
  - 증상: 인용문/카드 텍스트가 뷰포트 우측에 과도하게 붙음.
  - 해결: 모바일에서 히어로 타이틀·인용문 폰트 크기와 카드 패딩을 축소하여 여백을 확보.

- **로컬 서버 실행 시 샌드박스 권한 이슈**
  - 증상: 정적 서버 기동/스크린샷 캡처 명령이 샌드박스 제한으로 실패.
  - 해결: 필요한 권한으로 재실행하여 렌더링을 검증(데스크톱/모바일 스크린샷 확인, 모든 리소스 `200` 응답).

### 다국어 · 다크 테마 추가 작업 중 수정 사항

- **다크 테마에서 인버스 섹션 반전 문제 수정**
  - 증상: 핵심 컨셉·비전·셰어 카드가 `background: var(--ink)` + 하드코딩된 밝은 텍스트를 사용해, 다크 테마에서 `--ink`가 밝은 색으로 바뀌면 밝은 배경 위에 밝은 글자가 겹쳐 보이지 않게 됨.
  - 해결: 항상 어둡게 유지되어야 하는 패널을 전용 토큰(`--panel-bg`, `--panel-text`, `--panel-muted`, `--panel-border`)으로 분리하여 두 테마 모두에서 정상 표시.

- **다크 테마 네비게이션 / CTA 버튼 대비 문제 수정**
  - 증상: 네비 배경과 `nav__cta`가 밝은 색상 값(`rgba(240,238,230,...)`, `var(--ink)`)에 고정되어, 다크 테마에서 흰 배경·흰 글자로 깨짐.
  - 해결: `--nav-bg`, `--nav-bg-scrolled`, `--cta-bg`, `--cta-text` 토큰을 도입해 테마별로 값을 오버라이드.

- **테마 초기 적용 시 화면 깜빡임(FOUC) 방지**
  - 증상: 페이지 로드 후 JS가 실행되며 테마가 뒤늦게 적용되어 깜빡임 발생 가능.
  - 해결: `<head>`에 `localStorage`/`prefers-color-scheme`를 읽어 즉시 `data-theme`를 설정하는 인라인 스크립트를 추가.

- **모바일 네비 컨트롤 공간 정리**
  - 증상: 언어·테마 토글 추가로 모바일 헤더 폭이 부족.
  - 해결: 모바일 브레이크포인트에서 토글 버튼 패딩·폰트 크기와 컨트롤 간격을 축소해 한 줄에 정렬.
