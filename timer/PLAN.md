# 포모도로 타이머 및 할 일 목록 웹 애플리케이션 구현 계획 (PLAN.md)

이 문서는 프로젝트 폴더 내에 기록되는 설계 계획서입니다.

## 1. 구현 목표
- 포모도로 타이머와 할 일 목록(Todo List)이 유기적으로 동작하는 싱글 페이지 애플리케이션(SPA) 개발.
- 현대적인 UI/UX(다크 모드, 글래스모피즘, 반응형 레이아웃, 모드별 테마 변경) 적용.

## 2. 주요 기능 및 요구사항
- **포모도로 타이머**
  - 작업(Focus, 25분) / 짧은 휴식(Short Break, 5분) / 긴 휴식(Long Break, 15분) 모드.
  - 시간 카운트다운(시작, 일시정지, 리셋).
  - 타이머 종료 시 사운드 알림 (Web Audio API 사용, 브라우저 차단 우회를 위한 첫 인터랙션 시 활성화 방식 적용).
  - 각 모드에 따른 테마 색상(배경, 하이라이트 등)의 스무스한 전환.
- **할 일 목록 (Todo List)**
  - 할 일 등록 (엔터키 또는 추가 버튼).
  - 할 일 목록 표시 및 완료 여부 체크박스 토글.
  - 할 일 개별 삭제.
  - 진행도 게이지바 (전체 할 일 대비 완료된 할 일 비율 시각화).
  - 로컬 스토리지(`localStorage`)를 이용한 영구 저장 및 페이지 로드 시 복원.

## 3. UI/UX 디자인 가이드라인
- **색상 팔레트**:
  - 기본 배경: 딥 네이비/다크 퍼플 오로라 그라데이션
  - 글래스모피즘 카드: 투명도 있는 백그라운드(`rgba(255, 255, 255, 0.05)`) 및 블러(`backdrop-filter: blur(16px)`)
  - 작업 모드 컬러: Coral Red (`#ff6b6b`)
  - 짧은 휴식 모드 컬러: Mint Green (`#20c997`)
  - 긴 휴식 모드 컬러: Sapphire Blue (`#339af0`)
- **타이포그래피**: Outfit, Inter 구글 웹 폰트 로드.
- **애니메이션**:
  - 호버 효과: 버튼 스케일링, 박스 쉐도우 글로우.
  - 타이머 작동 시 부드러운 순환 진행 바(SVG Circle Progress) 또는 디지털 숫자 맥박 효과.
  - 할 일 추가/완료/삭제 시 트랜지션 처리.

## 4. 검증 계획
1. **타이머 기능**: 모드 변경 시 초기화 및 테마 전환 확인. 일시정지 후 재시작 시 시간 유지 확인.
2. **할 일 목록**: 데이터가 브라우저를 닫고 다시 열어도 로컬스토리지에 저장되는지 확인.
3. **반응형 테스트**: 모바일(가로폭 375px 내외)에서 레이아웃이 유연하게 맞추어지는지 확인.
4. **로컬 서버**: `python3 -m http.server 8000`으로 서빙 후 정상 동작 확인.
